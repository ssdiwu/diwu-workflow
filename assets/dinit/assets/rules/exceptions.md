# 异常处理与 BLOCKED 判定

> **规则约束级别说明**：本文件定义异常处理和 BLOCKED 判定规则。纠偏相关内容见 `correction.md`。除非特别标注 `[建议]`，否则都是必须遵守的约束。

## `<private>` 标签约定

用户用 `<private>...</private>` 包裹的内容（密码、密钥、个人敏感信息）不得写入 session 文件和 dtask。仅在当前对话上下文中使用，不得持久化。

## 需要人工介入的情况

| 类型 | 示例 |
|------|------|
| 缺少环境配置 | API 密钥、数据库连接、外部服务账号 |
| 外部依赖不可用 | 第三方 API 宕机、OAuth 授权、付费服务 |
| 验证无法进行 | 真实用户账号、未部署系统、特定硬件 |

## BLOCKED 判定

- **BLOCKED**：问题来自环境/凭据/权限/外部依赖不可用，任务需求不变
- 结论输出统一使用 `DECISION TRACE`（见 templates.md），再输出 BLOCKED 模板

### 判定锚点（现象→判断→动作）

#### 现象表

| 编号 | 现象 | 常见根因 |
|------|------|---------|
| F1 | 单元测试超时 | 资源不足/桩未就绪/用例缺陷 |
| F2 | API 返回 401 | 凭据缺失过期/权限不足/Token 过期 |
| F3 | 容器未运行 | 环境未启动/启动失败/端口映射错误 |

#### 判断矩阵

**BLOCKED（正例）**：问题来自环境/凭据/权限/外部依赖，任务本身无误。

| 现象 | 判断依据 | 结论 |
|------|---------|------|
| API 401 | 缺少凭据无法获取；acceptance 要求真实响应 | BLOCKED → InSpec + BLOCKED 模板 |
| 容器未运行 | acceptance 要求容器内验证；用户无权操作环境 | BLOCKED → InSpec + BLOCKED 模板 |
| 测试超时(资源型) | 用例无缺陷；其他环境可通过 | BLOCKED → InSpec + BLOCKED 模板 |

**保持 InProgress（反例）**：问题来自代码缺陷或实现错误。

| 现象 | 判断依据 | 结论 |
|------|---------|------|
| API 401 | 错误凭据/Token 拼接错误/请求头缺失 | 保持 InProgress，修复认证逻辑 |
| 容器未运行 | 配置文件有误（端口/路径/环境变量） | 保持 InProgress，修复配置 |
| 测试超时(用例缺陷) | 死循环/死锁/同步调用未 mock | 保持 InProgress，修复用例或加 mock |

**边界例**：存在替代验证路径。

| 现象 | 判断依据 | 结论 |
|------|---------|------|
| API 401 | 可用 mock/stub 替代；acceptance 未强制真实调用 | 保持 InProgress，记录风险 |
| 容器未运行 | 测试可直接本地运行无需容器 | 保持 InProgress，记录风险 |
| 测试偶发超时 | 不可复现；增加 timeout 后稳定 | 保持 InProgress，记录偶发风险 |

#### 动作表

| 结论 | 动作 |
|------|------|
| **BLOCKED** | 1.状态退回 InSpec 2.输出 DECISION TRACE 3.输出 BLOCKED 模板 4.停止等待人工 |
| **保持 InProgress** | 1.定位根因 2.修复 3.重新验证 4.session 记录修复过程 |

## 阻塞时的规则

**禁止**：提交 git commit / dtask 设为 Done / 假装完成
**必须**：session 记录进度和原因 / dtask 退回 InSpec（不是 InDraft）/ 输出 DECISION TRACE + BLOCKED 模板 / 停止等待人工

## 阻塞恢复流程

### 环境阻塞恢复
1. 人工提供配置/权限
2. Agent 验证环境可用
3. InSpec → InProgress
4. 继续实施
