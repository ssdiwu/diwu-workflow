# 异常处理

> **规则约束级别说明**：本文件定义异常处理的核心规则。除非特别标注 `[建议]`，否则都是必须遵守的约束。

## `<private>` 标签约定

用户用 `<private>...</private>` 包裹的内容（密码、密钥、个人敏感信息）不得写入 session 文件和 task.json。该内容仅在当前对话上下文中使用，不得持久化。

## 需要停止并请求人工帮助的情况

1. **缺少环境配置**:API 密钥、数据库连接、外部服务账号
2. **外部依赖不可用**:第三方 API 宕机、需人工授权的 OAuth、付费服务
3. **验证无法进行**:需要真实用户账号、依赖未部署的外部系统、需特定硬件

## BLOCKED 判定

- **BLOCKED**：问题来自环境、凭据、权限、外部依赖不可用，任务本身需求不变。
- 结论输出统一使用 `DECISION TRACE`（见 templates.md），再输出 BLOCKED 模板。

## BLOCKED 判定锚点

> **规则约束级别说明**：本章节定义判断是否为 BLOCKED 的决策锚点。判断矩阵基于「现象 → 判断 → 动作」框架，正例/反例/边界例作为排除项。

### 现象

| 编号 | 现象（证据） | 常见根因 |
|------|-------------|---------|
| F1 | 单元测试超时 | 测试环境资源不足、测试桩未就绪、测试用例本身缺陷 |
| F2 | API 返回 401 Unauthorized | 凭据缺失/过期、权限不足、Token 过期 |
| F3 | 容器未运行 | Docker/K8s 环境未启动、容器启动失败、端口映射错误 |

### 判断矩阵

**BLOCKED（正例）**：问题来自环境、凭据、权限、外部依赖不可用，任务本身需求不变。

| 现象 | 判断依据 | 结论 |
|------|---------|------|
| API 401 | 缺少凭据（API Key/OAuth Token），无法获取；任务 acceptance 要求真实 API 响应但无法满足 | BLOCKED：退回 InSpec，输出 BLOCKED 模板 |
| 容器未运行 | 任务 acceptance 要求容器内验证，但容器启动依赖外部环境（Docker/K8s）；当前用户无权操作 | BLOCKED：退回 InSpec，输出 BLOCKED 模板 |
| 单元测试超时（资源型） | 测试环境内存不足或 CPU 争抢，测试用例本身无缺陷；同一测试在其他环境可通过 | BLOCKED：退回 InSpec，输出 BLOCKED 模板 |

**保持 InProgress（反例）**：问题来自代码缺陷或实现错误，任务需求本身无误。

| 现象 | 判断依据 | 结论 |
|------|---------|------|
| API 401 | 代码中使用了错误凭据、Token 拼接错误、请求头缺失；修复代码即可解决 | 保持 InProgress：修复代码中的认证逻辑 |
| 容器未运行 | 容器配置文件有误（端口/路径/环境变量）；修复配置即可启动 | 保持 InProgress：修复容器配置 |
| 单元测试超时（用例缺陷） | 测试用例中存在死循环、死锁、或对外部系统的同步调用未 mock；测试本身需要修复 | 保持 InProgress：修复测试用例或添加 mock |

**边界例**：需进一步分析或存在替代验证路径。

| 现象 | 判断依据 | 结论 |
|------|---------|------|
| API 401 | 任务 acceptance 要求真实 API 响应，但可以通过 mock/stub 替代验证；acceptance 本身未强制要求真实调用 | 保持 InProgress：使用 mock 完成验证，记录风险 |
| 容器未运行 | 容器仅用于运行测试，但测试可通过本地命令直接运行（无需容器隔离） | 保持 InProgress：直接运行测试，记录风险 |
| 单元测试偶发超时 | 超时不可复现、可能与资源争抢相关；增加 timeout 重试后可稳定通过 | 保持 InProgress：增加 timeout 配置，记录偶发风险 |

### 动作

| 结论 | 动作 |
|------|------|
| **BLOCKED** | 1. 任务状态退回 InSpec；2. 输出 DECISION TRACE；3. 输出 BLOCKED 模板（见 templates.md）；4. 停止任务，等待人工介入 |
| **保持 InProgress** | 1. 定位根因（代码缺陷/配置错误/测试用例问题）；2. 修复后重新验证；3. 在 session 文件中记录修复过程 |

## 阻塞时的规则

**禁止**:
- 提交 git commit
- 将 `.claude/task.json` 状态设为 Done
- 假装任务已完成

**必须**:
- 在 session 文件中记录当前进度和阻塞原因
- 将 `.claude/task.json` 任务状态退回 **InSpec**(不是 InDraft)
- 输出 DECISION TRACE 后输出 BLOCKED 模板（见 templates.md），说明需要人工做什么
- 停止任务,等待人工介入

## 阻塞恢复流程

### 环境阻塞恢复
1. 人工提供所需配置/权限
2. Agent 验证环境可用
3. 任务状态: InSpec → InProgress
4. 继续实施
