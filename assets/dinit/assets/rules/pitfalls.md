# 误判防护体系

> **规则约束级别说明**：本文件定义误判防护的核心规则。除非特别标注 `[建议]`，否则都是必须遵守的约束。

## 三层架构

| 层次 | 内容 | 维护者 | 查阅时机 |
|------|------|--------|---------|
| **Layer 1** | 六类泛化模式（跨项目通用） | 本文件（内置） | 所有任务开工前必查 |
| **Layer 2** | 项目高频误判表（项目特有） | `.claude/project-pitfalls.md` | 出现异常时优先对照 |
| **Layer 3** | 接口预留（v2 扩展） | 未来版本 | 新误判类型发现后扩展 |

---

## Layer 1：六类泛化模式

很多误判可抽象为以下六类。看到对应现象时，**先查左列再下结论**。

### 1. 环境漂移

| 项 | 内容 |
|----|------|
| **现象** | 今天突然不通、502/404/超时、同提交多机表现不一致 |
| **默认先查** | 运行进程、端口占用、代理配置、环境变量、依赖服务状态 |
| **暂不下结论** | "代码回归了"、"上次还好好的" |
| **何时下结论** | 排除所有环境因素后，同一代码在干净环境中仍复现 |

### 2. 数据缺口

| 项 | 内容 |
|----|------|
| **现象** | 查询没命中、结果为空、候选缺失、显示"证据不足" |
| **默认先查** | 测试样本量、账号权限、数据存量、索引状态 |
| **暂不下结论** | "实现失效了"、"算法有问题" |
| **何时下结论** | 数据充足且权限正确时仍复现 |

### 3. 读层现象

| 项 | 内容 |
|----|------|
| **现象** | 页面白屏、布局错位、打不开、时间离谱、幽灵数据 |
| **默认先查** | 前端读层逻辑、渲染管线、开发服务状态、CSS/JS 加载 |
| **暂不下结论** | "后端主链挂了"、"API 返回错了" |
| **何时下结论** | 前端独立验证通过（直接调 API 确认返回正确）后仍有问题 |

### 4. 路由/护栏/契约

| 项 | 内容 |
|----|------|
| **现象** | tool-loop-guard 触发、空查询、选错工具、XML/格式漂移 |
| **默认先查** | 查询参数构造、工具路由规则、接口格式约定、护栏触发条件 |
| **暂不下结论** | "模型能力不行"、"这个模型做不到" |
| **何时下结论** | 参数和路由完全正确时仍复现 |

### 5. 验证误读

| 项 | 内容 |
|----|------|
| **现象** | 测试失败/脚本失败/HTTP 200/成功提示/撤回成功文案 |
| **默认先查** | 测试口径和前提、脚本执行时机、表面提示含义、首轮输出完整性 |
| **暂不下结论** | "真实失败了" 或 "真实成功了" |
| **何时下结论** | 用 L1-L3 证据交叉验证后（见 verification.md） |

### 6. 分层未拆清

| 项 | 内容 |
|----|------|
| **现象** | 结构化结果差、产出回退、改了没效果 |
| **默认先查** | 输入层 → 提示词层 → 后置判定层 → 回退层，逐层隔离 |
| **暂不下结论** | "提示词的问题" 或 "代码的问题"（单一归因） |
| **何时下结论** | 逐层排查后锁定单一层，且其他层独立验证正常 |

## Layer 2：项目高频表机制

| 项 | 说明 |
|----|------|
| **存储位置** | `.claude/project-pitfalls.md`（用户项目本地维护） |
| **生成方式** | 人工积累或 `/dinit` 从 `project-pitfalls.md.template` 初始化 |
| **消费场景** | 出现异常时，先查 Layer 1 泛化模式 → 再查本项目高频表 → 再做判断 |
| **格式** | 与 Layer 1 一致的四列表格（现象/先查/不下结论/何时下结论） |
| **更新时机** | 每次纠偏后发现新的项目特有误判时追加 |

## 内置插件开发高频误判（示例）

以下 2-3 条为本项目（diwu-workflow 插件开发）的高频误判示例，供参考格式：

| 现象 | 默认先查 | 暂不下结论 |
|------|---------|-----------|
| hooks.json 格式正确但不触发 | hook 注册路径、事件名称大小写、Claude Code 版本兼容性 | "插件坏了"、"hook 语法有误" |
| rules 注入了但 Agent 行为不变 | rules-manifest.json 是否包含新文件、hook 注入顺序、缓存未刷新 | "规则写了没用"、"Agent 不听指令" |
| JSON 校验通过但功能异常 | 字段拼写（大小写）、嵌套结构深度、必填字段遗漏 | "逻辑有 bug" |
