# /djug — 判断决策

按 djug skill 的判断锚点框架，输出 DECISION TRACE。

## 触发方式

- `/djug` — 列出所有判断锚点速查
- `/djug 是否拆分任务` — 针对具体问题应用判断框架
- `/djug 大幅度判定` — 查询特定判断维度

## 四段式判断索引

### 一、启动判断

#### 基线失败时先修基线还是继续
- **正例**：smoke.sh build/lint 失败 → 先修复基线
- **反例**：明知失败仍继续开发 → 违规

#### 不确定性决策
- **直接做**：改动小 + 结果可预期 + 差异一句话说清
- **先写最小规格**：结果说不清 / 依赖外部系统 / 第一次做
- **先探索验证**：外部依赖多 / 组合行为不稳定 / 回滚成本高

### 二、实施判断

#### 大幅度修改判定
- **正例**：接口字段变更 / 改动 > 2000 行 → 输出 REVIEW 等人工确认
- **反例**：修复内部 bug / 改动 < 200 行 → 自审后 Done

#### 执行偏差分级
- **Level 2**：缺少 email 校验等明显遗漏 → 自动补充
- **Level 4**：换数据库等架构变更 → 停下问用户

#### 并行与串行
- **并行**：子代理 A 改 src/auth/，B 改 src/payment/，无共享文件
- **串行**：两个代理都要改同一文件或同一迁移文件

### 三、验收判断

#### InReview → Done 是否需人工确认
- **正例**：API/字段变更 / 改动 > 2000 行 → 人工确认
- **反例**：小幅度修改 + acceptance 全过 → 自审后 Done

#### blocked_by 何时不该写
- **该写**：依赖未 Done 任务生成的产物
- **不该写**：调用已稳定函数且前置任务已 Done

### 四、纠偏判断

#### continuous_mode 续跑
- `true`：Done 后自动选下一任务
- `false`：每完成一个任务停止，输出摘要

## DECISION TRACE 输出格式

```
**结论**: [决策结果]
**现象**: [观察到什么]
**判断锚点**: [命中了哪个规则]
**红旗信号**: [如果有]
```
