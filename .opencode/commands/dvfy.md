# /dvfy — 验证规则

快速查询 dvfy skill 的验证规则，辅助判断任务是否达到 Done 标准。

## 触发方式

- `/dvfy` — 输出验证规则摘要
- `/dvfy 任务ID` — 检查指定任务的验收条件是否充分
- `/dvfy L1` — 查询特定证据等级的验证方法

## L1-L5 五级证据

| 等级 | 名称 | 说明 | 示例 |
|------|------|------|------|
| **L1** | 运行态证据 | 真实输出已发生可识别变化 | 请求返回值变了、页面渲染变了 |
| **L2** | 调用链证据 | 底层状态切换已被观测到 | 日志命中新路径、数据库已写入 |
| **L3** | 自动化断言 | 测试/脚本自动判定 | 单元测试全绿、verify.sh exit 0 |
| **L4** | 表面观察 | 需人工解读的现象 | 截图看起来正确、日志无报错 |
| **L5** | 间接推断 | 代码 diff / 口头说明 | "我改了这段代码" |

**默认优先级**：L1-3 主判 → L4-5 辅助。只有 L5 时不得宣称完成。

## Done 判定门槛

| 证据组合 | 判定动作 |
|---------|---------|
| 全部 L1-3 通过 | Agent 自审后 Done |
| L1-3 部分 + L4 补充 | 自审后 Done，L4 项标注「待人工确认」|
| 仅 L4 或 L4-5 | **不允许 Done**，保持 InReview |
| 仅 L5 | **禁止宣称完成**，保持 InProgress |

## 完成前四问

标记 InReview 前必须回答：

1. **验证的是哪一层？** 文档 / 代码 / 配置 / 调用链 / 运行态 / 输出
2. **是否真的走到了新链路？** 是/否 + 证据
3. **结果是否出现可识别变化？** 变化描述 + 前后对比
4. **哪些已验证，哪些未验证？** 已验证清单 + 未验证清单

## 无法验证时的三段式

```text
未验证项: [具体条目]
原因: [为什么现在无法验证]
后续验证方式: [具体步骤，不是"以后再测"]
```

## 运行态验证方法

| 方法 | 适用场景 |
|------|---------|
| 关键点加日志/断言 | 有源码访问权限 |
| 发起真实请求 | API / CLI / Web 服务 |
| 检查产物生成 | 构建任务 / 插件项目 |
| 端到端走查 | UI / 多模块集成 |
